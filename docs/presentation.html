Firebase Auth
User Registration Flow
  • Input ValidationName, Email, Password validated before sending.
  • createUserWithEmailAndPassword()Firebase creates account — password is hashed, never stored in plain text.
  • Profile Auto-CreatedsaveUserProfile(uid) writes users/{uid} doc with name, email, points: 0.
  • Auth State ChangeSDK triggers state change → app navigates to Art List automatically.
Firebase Auth · JWT
Login & JWT Token Flow
  • signInWithEmailAndPassword()Firebase verifies credentials server-side, returns FirebaseUser.
  • JWT ID TokenSigned token with UID, email, expiry — auto-attached to every Firestore request as Bearer token.
  • Auto-refreshSDK silently renews token every hour, no user interaction required.
  • Security RulesFirestore verifies request.auth.uid from JWT before granting access.

Quality Assurance
Testing — Strategy & Process
  • Unit Tests (JVM)Run on the JVM — no emulator needed, instant feedback. Test pure business logic: points calculation, badge boundaries.
  • Instrumented UI TestsRun on emulator/device via connectedAndroidTest. Use ComposeTestRule to interact with real Compose UI.
  • Hilt in TestsHiltTestRunner replaces the default runner — starts HiltTestApplication which enables DI injection in test classes annotated with @HiltAndroidTest.
  • HiltTestActivityA debug-only empty Activity that acts as the host for Compose content during UI tests, avoiding the need to launch the full app.
  • Run Commands./gradlew test for unit tests · ./gradlew connectedAndroidTest for UI tests.

Accessibility · Android
Accessibility — Types & How They Work
  • TalkBack (Vision)Screen reader built into Android. Reads every UI element aloud using contentDescription from the Compose Semantics tree. Blind users navigate by swiping.
  • Touch Targets (Motor)WCAG 2.1 requires ≥ 48×48dp for all interactive elements. In Compose: Modifier.size(48.dp) or .padding() to inflate tap area.
  • Color Contrast (Vision)WCAG AA: text contrast ≥ 4.5:1. White (#FFF) on our purple (#6200EE) achieves ~5.9:1 — passes.
  • Text Scaling (Vision)Using sp units (not dp) for fonts allows text to scale with the system "Font Size" setting without breaking layouts.
  • Semantics (All)Compose builds a parallel Semantics Node Tree used by TalkBack, Switch Access, and UI test framework alike.
Accessibility · Implementation
Accessibility in ArtFinder
  • contentDescription on ImagesAsyncImage(contentDescription = artwork.title) — TalkBack announces artwork name instead of "image".
  • Icon ButtonsIconButton { Icon(contentDescription = "Back") } — navigation icons have meaningful labels for screen readers.
  • TextField LabelsOutlinedTextField(label = { Text("Email") }) — label stays visible; TalkBack reads it as field hint.
  • Error MessagesValidation errors shown inline below the field — not just red color, always accompanied by text.
  • Layout InspectorVerified with Android Studio's Accessibility Scanner: no missing descriptions, sufficient touch targets.
