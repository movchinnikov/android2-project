Architecture
MVVM + Repository Pattern
  • UI LayerJetpack Compose screens observe StateFlow from ViewModels.
  • ViewModelHolds UI state, calls repositories, no direct data access.
  • RepositorySingle source of truth. Merges API + Firestore data.
  • DIDagger Hilt for all dependencies.
Data Persistence · Cloud
Auth & User Profiles
  • Firebase AuthEmail/password registration and login via AuthRepository.
  • Auto-created ProfileOn first register, a user document is written to Firestore users/{uid}.
  • Profile UpdatesEdit name, change password — synced to Firestore in real time.
  • Persistent SessionFirebase SDK persists auth state; app skips login screen on relaunch.

Firebase Auth
User Registration Flow
  • Input ValidationName, Email, Password validated before sending.
  • createUserWithEmailAndPassword()Firebase creates account — password is hashed, never stored in plain text.
  • Profile Auto-CreatedsaveUserProfile(uid) writes users/{uid} doc with name, email, points: 0.
  • Auth State ChangeSDK triggers state change → app navigates to Art List automatically.
Firebase Auth · JWT
Login & JWT Token Flow
  • signInWithEmailAndPassword()Firebase verifies credentials server-side, returns FirebaseUser.
  • JWT ID TokenSigned token with UID, email, expiry — auto-attached to every Firestore request as Bearer token.
  • Auto-refreshSDK silently renews token every hour, no user interaction required.
  • Security RulesFirestore verifies request.auth.uid from JWT before granting access.
User Profile
Profile Screen
  • Reads from FirestoreOn load, ProfileViewModel fetches the user document from users/{uid}.
  • Displays Points & BadgeTotal accumulated points and current achievement badge (Explorer / Curator / Archivist) shown with icon.
  • Edit NameNavigates to EditNameScreen, calls UserRepository.updateProfileName() to persist change.
  • Leaderboard LinkNavigates to LeaderboardScreen which reads all users from Firestore and ranks them by points.
  • LogoutCalls FirebaseAuth.signOut(), clears state and navigates back to Login.

Networking · API
Retrofit — Under the Hood
  • Interface → ProxyRetrofit generates a dynamic proxy at runtime that implements your ArtApiService interface.
  • OkHttp TransportActual HTTP connection, connection pooling, disk cache, and logging interceptors.
  • Coroutines Adaptersuspend functions run on IO thread — result delivered on Main thread safely.
  • Gson ConverterJSON field "artist_display" maps to Kotlin artist_display: String? automatically.
  • Pagination?page=1&limit=20 query params via @Query annotations.
Location Services · Maps SDK
Google Maps — How Markers Work
  • Runtime PermissionsrememberLauncherForActivityResult requests ACCESS_FINE_LOCATION on map open.
  • Gallery MarkersLoaded from AIC API /galleries, filtered to those with coordinates.
  • Marker Colors🟢 Green = visited, 🔴 Red = not visited, 🟣 Violet = gallery cluster, 🔵 Azure = single gallery.
  • ClusteringNearby galleries grouped by rounded coordinates (×3000 precision).
  • Bottom SheetTap gallery cluster → ModalBottomSheet with list and thumbnails.
